🎯 1. OBJETIVO DE LA REVISIÓN
Transformar la persistencia de datos de un modelo de "entorno de desarrollo" a un modelo de "aplicación de sistema", garantizando la privacidad del usuario y la independencia del código fuente.

🛠️ 2. HITOS TÉCNICOS ALCANZADOS
📂 A. Externalización de Rutas (Anexo I)
Se ha eliminado la dependencia de la carpeta local /storage. Ahora el programa se integra con el Sistema Operativo.

Lógica: Uso de System.getProperty("os.name") para detectar el entorno.

Rutas Dinámicas: * Windows: %AppData%/Habitly/

Unix/Mac: ~/.habitly/

Decisión de Arquitectura: Centralización en AppConfig para desacoplar la lógica de rutas del GestorInventario.

🔐 B. Blindaje Criptográfico AES-128 (Anexo II)
Los datos han dejado de ser binarios simples para ser cifrados de extremo a extremo.

Algoritmo: AES (Advanced Encryption Standard) con bloques de 128 bits.

Key Derivation (Llave Fantasma): La clave no reside en el código. Se genera en tiempo de ejecución combinando el user.name y la arquitectura del sistema, procesados mediante un hash SHA-256.

Impacto: Los datos son ilegibles fuera del ordenador original del usuario.

🧹 C. Sanitización de Datos (Anexo III)
Implementación de un protocolo de "Borrado de Fábrica".

Persistencia: Eliminación física del archivo .dat mediante java.nio.file.Files.

Memoria: Limpieza inmediata del ArrayList en RAM para evitar fugas de información tras el borrado.

📝 3. ESQUEMA DE CLASES MODIFICADAS
Clase,                Responsabilidad en v1.0.1A
AppConfig,            Gestiona constantes de negocio (IGIC) y provee rutas de sistema seguras.
CryptoManager,        Motor de cifrado. Genera la llave dinámica y procesa byte[].
GestorInventario,     Orquestador de persistencia. Une la serialización con el cifrado.
GestionInmobiliaria,  Interfaz de usuario. Implementa el menú de seguridad y confirmaciones.

4. NOTAS PARA EL FUTURO (Backlog)
Migración de Llave: Si el usuario cambia de PC, los datos no serán recuperables debido a la vinculación con el Hardware ID. Se debería considerar un sistema de "Exportación/Importación" con contraseña manual.

Logs: Sustituir los System.out.println de sistema por un Logger profesional para evitar ruido en la consola del usuario.

Estado del Proyecto: 🟢 ESTABLE > "Los datos ya no pertenecen al proyecto, pertenecen al usuario y están protegidos por el sistema."
